Pros and Cons of Becoming a Software Engineer

Pros

1. High Pay

2. High Demand

3. Control

4. Flexibility

a. Remote Work

b. Lots of options for work depending on what you lke

- Work for a big company

- Work for a startup

- Work for yourself

You may enjoy Software Engineering if you:

- Enjoy building things that other people will use and benefit from

- Enjoy coming up with solutions to problems

- Enjoy being creative

- Enjoy working on challenging problems

- Enjoy math, puzzles, problem solving, and other kinds of analytical tasks

- Enjoy learning/doing new things every day



Cons

1. Takes lots of hard work and patience to get really good at it

2. Technology is always changing. You need to continuously learn new things

You may not enjoy Software Engineering if you:

- Don't enjoy math, puzzles, problem solving, and other kinds of analytical tasks

- Don't enjoy learning new things

- Are uncomfortable with not knowing things

- Can't tolerate being stuck on a problem

About the course

How it works

1. The course is set up to help you learn things and build a portfolio at the same time. Towards the end we'll work on interview and job search skills. We'll do the first 4 sessions before Ramadan. It'll mostly be front end development. If that goes well we can do more advanced topics after Ramadan.


2. Every class you'll get a project and we'll go over it the next class. You'll have a project due for the first class.


3. Because you need to learn how to learn things on your own, a lot of this will depend on you doing the projects in your own time. During classes, I'll give you the basics and review the previous session's project.


How to succeed

1. Do all of the projects

2. Expect to spend at least 10-12 hours a week

3. If you get stuck, ask for help

4. And most importantly, be open to constantly learning new things



The #1 skill to work on to become a successful software engineer:

The ability to quickly learn and implement new things

This is something that can be learned with practice
